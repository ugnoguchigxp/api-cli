# apicli アーキテクチャ

## スコープ

`apicli` は、外部の REST API との連携を目的とした TypeScript サーバーパッケージです。

このパッケージは、インバウンドのサーバー認証（JWT、セッションなど）を管理しません。

## コンポーネント

- `ApiClient`
  - リクエストの実行
  - タイムアウト処理
  - レスポンスのパース（`auto`/`json`/`text`/`raw`）
  - エラーの正規化（`ApiCliError`, `ApiCliHttpError`）
- `ProviderResolver`
  - リクエストやテナントごとの実行時プロバイダ解決
- `AuthAdapter`
  - 既存のサーバー認証・セッション・トークン層を利用した、アウトバウンド認証ヘッダーの注入
- `コンテキストヘルパー` (context helpers)
  - フレームワークごとのリクエスト型間で、ヘッダーから抽出したコンテキストを正規化
- `プロキシサーバーモード`
  - `createApiProxyServer` で設定駆動の共通ハンドラを生成
  - Express/Nest/Fastify/Hono/Next.js の導入コードを最小化

## スコープ外（Non-goals）

- フレームワーク固有のDI（依存性注入）やコンテナへの統合
- ローカルのキーボルトや SQLite 等によるデータ永続化
- サービスレベルにおけるユーザー認証の代替

## 拡張ポイント

- `ApiClient` 外部でのリトライやサーキットブレーカーのラップ処理
- `client.call` 実行時のカスタムテレメトリフック
- テナントを考慮したプロバイダの解決およびトークン取得ロジック
